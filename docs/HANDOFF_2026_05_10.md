### ★ AGREED 2026-05-11: implement static-prior gating (paper architectural contribution)

Goal: add a learnable mixing weight between `p_model` and `p_climatology`
to give the paper a real "architectural innovation" beyond a vanilla
transformer. Backstop: post-hoc α=0.4 blend already gave 7.83→8.63
(+10%). A learnable gate's lower bound is that number.

**Math**:
```
gate    = sigmoid(W_gate · decoder_features)         # [0,1] per pixel/lead
p_final = gate * p_model + (1-gate) * p_clim_patch
loss    = focal_BCE(p_final, label)                  # gradient flows to gate
```

**Run all 4 ablation variants** (paper-quality gradient):
| Variant | gate shape | params added |
|---|---|---|
| A. baseline (no gate) | — | 0 |
| B. global α | scalar | 1 |
| C. per-lead α | (n_leads,) ≈ 32 | 32 |
| D. per-pixel per-lead | (B, L, P, 1) via Linear(d_model→1) | ~256 |

Each on 12y enc21 9ch (matches single-ckpt SOTA setup), 6h GPU each → 24h total.

**Concrete code changes** (write in next window):
1. `src/training/s2s_hotspot.py` — add `gating_mode` arg to `S2SHotspot.__init__`, branch in `forward()` to mix p_model with p_clim_patch from decoder_ctx
2. `src/training/train_v3.py` — add `--gating {none,global,per_lead,per_pixel}` flag; pass fire_clim_per_patch from decoder_ctx through to model.forward
3. `slurm/train_v3_gating_sweep_narval.sh` — new wrapper, env-var GATING=...
4. Submit 4 jobs (one per variant), then save_scores + metric_card on each
5. Add ablation table to RESULTS_TABLE

**Time**: ~3h coding + 24h GPU + 2h analysis.

**Why this specifically**: of the 3 architectural-novelty candidates
(variable-tokenization / lead-conditional adapter / static-prior gating),
this one has the best ROI — lowest difficulty, highest expected ΔLift
because of the post-hoc backstop, and the cleanest paper story.
